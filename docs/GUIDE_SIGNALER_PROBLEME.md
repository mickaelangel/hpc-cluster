# Guide pour Signaler un Problème ou Demander une Fonctionnalité

## 🐛 Signaler un Bug

### Avant de Signaler

1. **Vérifier la documentation** : Consultez [docs/GUIDE_TROUBLESHOOTING.md](GUIDE_TROUBLESHOOTING.md)
2. **Chercher les issues existantes** : Vérifiez si le problème a déjà été signalé
3. **Vérifier les versions** : Assurez-vous d'utiliser la dernière version

### Comment Signaler

1. Aller sur : https://github.com/mickaelangel/hpc-cluster/issues/new?template=bug_report.md
2. Remplir le formulaire avec :
   - **Description claire** du problème
   - **Étapes pour reproduire** le bug
   - **Comportement attendu** vs **comportement observé**
   - **Informations système** (OS, versions, etc.)
   - **Logs** si disponibles

### Informations à Fournir

- **OS** : openSUSE 15.6, etc.
- **Version Docker** : `docker --version`
- **Version du projet** : Commit ou tag Git
- **Logs** : Messages d'erreur complets
- **Configuration** : Fichiers de config pertinents (sans secrets)

## ✨ Demander une Fonctionnalité

### Avant de Demander

1. **Vérifier les issues existantes** : La fonctionnalité a peut-être déjà été proposée
2. **Vérifier la roadmap** : Voir la section Roadmap du README
3. **Décrire clairement** : Expliquer le besoin et l'usage

### Comment Demander

1. Aller sur : https://github.com/mickaelangel/hpc-cluster/issues/new?template=feature_request.md
2. Remplir le formulaire avec :
   - **Description** de la fonctionnalité souhaitée
   - **Problème résolu** : Quel problème cette fonctionnalité résout-elle ?
   - **Solution proposée** : Comment devrait-elle fonctionner ?
   - **Alternatives** : Solutions de contournement actuelles

---

## 📋 Bonnes Pratiques

### Pour les Bugs

✅ **À faire** :
- Décrire clairement le problème
- Fournir les étapes de reproduction
- Inclure les logs d'erreur
- Spécifier l'environnement

❌ **À éviter** :
- Signaler sans avoir vérifié la documentation
- Ne pas fournir d'informations système
- Inclure des mots de passe ou secrets

### Pour les Fonctionnalités

✅ **À faire** :
- Expliquer le besoin métier
- Décrire l'usage prévu
- Proposer une solution
- Vérifier si c'est dans la roadmap

❌ **À éviter** :
- Demander sans expliquer le besoin
- Proposer des changements majeurs sans discussion
- Ignorer les contraintes techniques
